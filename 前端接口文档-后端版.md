# 前端接口文档（后端版）

> 本文档面向后端开发，列出前端当前所有真实/Mock 接口的调用方式、请求字段及期望响应字段。Base URL 统一为 `/api`。

---

## 1. 基础约定

| 项 | 说明 |
|----|------|
| Base URL | `/api`（Vite 代理到 `http://10.11.20.16:8080`） |
| 认证方式 | `Authorization: Bearer {token}` |
| 响应格式 | `{ code: 200, data: ..., message: ... }`，前端 `request.js` 自动取 `data.data` |
| 成功码 | `200` |
| 认证失败 | `401`，前端清空 token 并跳转登录 |
| Mock 开关 | `.env` 中 `VITE_USE_MOCK=true` 时走本地 Mock；联调时改为 `false` |

---

## 2. 异步任务机制

生成类接口采用**提交任务 → 轮询状态 → 取结果**三步骤：

1. `POST` 生成接口返回 `{ taskId }`；
2. 前端轮询 `GET /tasks/{taskId}`，期望返回：

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | string | `SUCCESS` / `FAILED` / `CANCELED` / `RUNNING` / `PENDING` |
| `resultId` | string | 成功时用于下一步查询结果 |
| `errorMessage` | string | 失败时错误信息 |
| `progress` | number | 可选，任务进度 0-100 |

3. 成功后根据 `resultId` 调 `GET /xxx/{resultId}` 获取最终结果。

默认轮询间隔 `3s`，超时 `120s`。

## 8. 实时调控（Realtime Control）

### 8.1 导入实时调控 JSON

- **METHOD**：`POST`
- **PATH**：`/realtime-control/import`

**参数（Query）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `controlDate` | string | 是 | 调控日期 `yyyy-MM-dd` |
| `sourceFileName` | string | 否 | 源文件名，默认 `realtime_control.json` |

**请求体（Body）：**

支持两种结构：

**结构 A（算法组新 JSON）：**

```json
{
  "timestamp": "14:32:18",
  "control": {
    "boiler_load": 28.5,
    "turbine_output": 250.0,
    "steam_pressure": 3.5,
    "power_factor_target": 0.95,
    "grid_purchase": 1200
  },
  "forecast": {
    "elec_next_5min": 1280,
    "steam_next_5min": 46.2
  }
}
```

**结构 B（扁平旧结构）：**

```json
{
  "boilerLoad": 28.5,
  "turbineOutput": 250.0,
  "steamPressure": 3.5,
  "powerFactorTarget": 0.95,
  "gridPurchase": 1200,
  "elecDemand": 1280,
  "steamDemand": 46.2
}
```

**响应（data）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | boolean | 是否成功 |
| `controlId` | number/string | 调控记录 ID |

### 8.2 获取最新 MPC 调控结果

- **METHOD**：`GET`
- **PATH**：`/realtime-control/latest`

**响应（data）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `controlId` | number/string | 调控 ID |
| `controlDate` | string | 调控日期 |
| `timestamp` | string | 时间戳 |
| `boilerLoad` | number | 锅炉负荷（t/h） |
| `turbineOutput` | number | 汽机出力（kW） |
| `gridPurchase` | number | 外购电力（kW） |
| `powerFactorTarget` | number | 功率因数目标 |
| `elecNext5min` / `elecDemand` | number | 未来 5min 电力需求（kW） |
| `steamNext5min` / `steamDemand` | number | 未来 5min 蒸汽需求（t/h） |
| `createTime` | string | 创建时间 |

### 8.3 获取调控历史

- **METHOD**：`GET`
- **PATH**：`/realtime-control/history`

**参数：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `pageNum` | number | 否 | 页码，默认 `1` |
| `pageSize` | number | 否 | 每页条数，默认 `10` |
| `startDate` | string | 否 | 开始日期 `yyyy-MM-dd` |
| `endDate` | string | 否 | 结束日期 `yyyy-MM-dd` |

**响应（data）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `records` | array | 调控记录列表，字段同 8.2 |

## 12. 附录：前端请求工具

`src/utils/request.js` 封装了 axios：

- 自动带 `Bearer token`；
- 统一处理 `code !== 200`；
- 对 `401` 自动跳转登录；
- 返回 `response.data.data`。

因此后端只需保证响应格式为 `{ code: 200, data: ..., message: ... }` 即可。
